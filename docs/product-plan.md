# Mac 瘦身管家产品规划

## 产品定位

Mac 瘦身管家是一个本地运行的 macOS 存储分析与安全清理工具。第一目标不是尽可能多地删除文件，而是帮助用户看懂“系统数据”主要由什么构成，并在可控范围内清理低风险缓存。

## 核心目标

1. 图形化展示系统数据相关空间占用。
2. 当系统数据相关占用超过阈值时提醒用户。
3. 支持用户选择清理项，并执行安全的一键瘦身。

## 第一版 MVP 范围

### 必做

- 仪表盘
  - 展示总可疑系统数据占用。
  - 展示各分类占比。
  - 展示可释放空间预估。
  - 展示最近一次扫描时间。

- 分类扫描
  - 浏览器缓存。
  - App 缓存。
  - 系统日志与诊断文件。
  - 下载目录大文件。
  - 废纸篓。
  - 开发环境缓存，优先识别 Xcode 缓存。

- 风险等级
  - 低风险：默认可清理。
  - 中风险：需要用户手动勾选。
  - 高风险：只展示，不进入一键清理。

- 一键瘦身
  - 默认只勾选低风险项。
  - 提供安全清理 / 深度清理两种模式。
  - 清理前展示确认弹窗。
  - 清理后展示释放空间和失败项。

- 提醒
  - 默认阈值为 150GB。
  - 阈值支持在设置页调整。
  - 当扫描结果超过阈值时，在 App 内展示提醒。
  - 用户开启桌面通知后，超过阈值时发送 macOS 通知。

### 暂不做

- 自动后台清理。
- 清理系统关键目录。
- 清理用户文档、照片、桌面文件。
- 深度卸载 App。
- 云盘清理。
- 多用户账号扫描。

## 缓存分类

| 分类 | 典型路径 | 风险 | 第一版动作 |
| --- | --- | --- | --- |
| 浏览器缓存 | `~/Library/Caches/Google/Chrome`, `~/Library/Caches/com.apple.Safari`, `~/Library/Caches/Microsoft Edge` | 低 | 可清理 |
| App 缓存 | `~/Library/Caches/*` | 低到中 | 默认只清理明确缓存目录 |
| App 残留 | `~/Library/Application Support/*`, `~/Library/Preferences/*` | 中到高 | 第一版只展示 |
| 系统日志 | `~/Library/Logs`, `/Library/Logs` | 低到中 | 用户确认后清理 |
| 诊断文件 | `~/Library/Logs/DiagnosticReports` | 低 | 可清理 |
| 下载大文件 | `~/Downloads` | 高 | 只展示 |
| 废纸篓 | `~/.Trash` | 中 | 用户确认后清理 |
| Xcode 缓存 | `~/Library/Developer/Xcode/DerivedData` | 中 | 用户确认后清理 |

## 安全规则

1. 第一版不删除系统根目录下的关键文件。
2. 不删除用户文档类文件，除非用户明确选择。
3. 不清理 `~/Library/Application Support` 中无法确认归属的目录。
4. 每个清理动作必须记录日志。
5. 清理前必须显示预计释放空间。
6. 清理失败时保留错误原因。

## 页面结构

### 仪表盘

- 顶部：总占用、可释放空间、风险提醒。
- 中部：分类环形图和占比条。
- 下方：分类列表，每项展示名称、大小、风险、是否可清理。
- 底部：扫描按钮、一键瘦身按钮。

### 扫描结果页

- 展示各分类明细。
- 支持展开查看路径。
- 支持选择清理项。

### 设置页

- 系统数据提醒阈值。
- 是否启用桌面通知。
- 是否扫描开发者缓存。
- 排除路径列表。

### 清理报告页

- 本次释放空间。
- 成功清理项。
- 失败项。
- 下一步建议。

## 用户流程

1. 用户打开 App。
2. App 展示上次扫描结果，没有结果时提示开始扫描。
3. 用户点击“扫描”。
4. App 按分类统计候选目录大小。
5. App 展示仪表盘和分类列表。
6. 如果总占用超过阈值，展示提醒。
7. 用户勾选想清理的项目。
8. 用户点击“一键瘦身”。
9. App 二次确认。
10. App 执行清理并生成报告。
